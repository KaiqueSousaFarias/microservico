# Desafio Microservices - Order + Payment

Este repositório contém dois serviços que devem se comunicar:

- Order Service (responsável por pedidos)
- Payment Service (responsável por pagamentos)

---

## Objetivo

Desenvolver dois serviços independentes utilizando Java e Quarkus que se comunicam via HTTP.

---

## Estrutura

- /order-service → API de pedidos
- /payment-service → API de pagamentos
